# シナリオ A — 確認フロー + Human検索

**対象 US**: US-1-3（確認フロー）、US-10（Human検索）  
**検索モード**: human（デフォルト）  
**前提条件**: ブラウザが使用可能であること

---

## 事前確認：ENV設定

`.env` の `SEARCH_ENGINE_URL` が Google を向いていることを確認する。

```env
SEARCH_ENGINE_URL=https://www.google.com/search?q=
```

テスト用ローカルサーバーが設定されている場合は上記に書き換える。

---

## 起動

```bash
python -m research_team.cli.main start --depth standard
```

ブラウザが自動で開き、Research Team Control Panel が表示される。

---

## A-1: 初期メッセージ確認

**操作**: 起動後、何も入力せず待つ

**期待結果**:
- チャット欄（左ペイン）に CSM からメッセージが表示される
- メッセージ内容: 「こんにちは！リサーチするテーマを入力してください。」

---

## A-2: テーマ入力

**操作**: チャット欄に以下を入力して「送信」ボタンを押す（または Enter キー）

```
Pythonの歴史
```

**期待結果**:
- 入力したメッセージが右寄りのバブル（紫背景）で表示される
- CSM から確認メッセージが返る
- 確認メッセージに **「Pythonの歴史」** と **「標準」**（または `standard`）の両方が含まれる
- 確認メッセージ例: 「テーマ「Pythonの歴史」、深さ「標準」で調査します。よろしいですか？（はい／いいえ）」
- この時点では右ペインの進捗ログは **動いていない**（調査未開始）

---

## A-3: 承認

**操作**: チャット欄に以下を入力して送信

```
はい
```

**期待結果**:
- 「「Pythonの歴史」の調査を開始します。チームを編成しています…」のメッセージが CSM から届く
- 右ペインの **進捗ログ**（右下）に `→ テーマ: Pythonの歴史` 等が流れ始める
- 右ペインの **エージェント出力**（右上）に PM またはチームビルダーのテキストが流れ始める

---

## A-4: Human検索 — 1回目の取り込み操作

**タイミング**: 進捗ログに `→ 調査員 が処理中...` または `🔍 調査員: web_search 「...」` が出たあと

**操作の流れ**:

1. 右ペイン下部に **承認バナー** が出現する
   - 青枠のパネルに URL と「✅ 取り込む」「❌ スキップ」ボタンが表示される
2. バナーに表示された URL をブラウザの別タブで開き、ページが正しくロードされていることを確認する
3. 内容を調査に使いたい場合 → **「✅ 取り込む」** を押す  
   内容が不要・CAPTCHAなどで取得不能な場合 → **「❌ スキップ」** を押す

**期待結果（取り込む を押した場合）**:
- バナーが消える
- 進捗ログに `✓ 調査員: web_fetch 完了` または `✓ web_search 完了` が表示される
- 続けて次の URL の承認バナーが出る（検索結果の件数分繰り返す）

**期待結果（スキップ を押した場合）**:
- バナーが消える
- 進捗ログに次の処理が続く（そのURLはスキップされた扱い）

---

## A-5: Human検索 — 繰り返し操作

スペシャリストエージェントは複数の `web_search` / `web_fetch` を実行する。  
承認バナーが出るたびに A-4 の操作を繰り返す。

**目安**: 1エージェントあたり 3〜10 回程度バナーが出る。複数エージェントが順次実行される。

**判断基準（取り込む vs スキップ）**:
- テーマ「Pythonの歴史」に関連するページ → 取り込む
- CAPTCHA / アクセス拒否 / 明らかに無関係 → スキップ

---

## A-6: 完了確認

**タイミング**: 承認バナーが出なくなり、進捗ログの更新が止まったあと

**期待結果**:
- CSM から「調査が完了しました（品質スコア: X.XX）。\n出力: /path/to/report.md」のメッセージが届く
- 進捗ログ最下部に `✓ 完了: /path/to/report.md` が表示される
- 指定されたパスに Markdown ファイルが生成されている

**確認方法**:

```bash
# 出力ファイルを開く（パスはUIに表示されたものに置き換える）
start /path/to/report.md
```

## 合否基準まとめ

| # | 確認項目 | 合格条件 |
|---|---|---|
| 1 | 初期メッセージ | CSM からテーマ入力を促すメッセージが出る |
| 2 | 確認メッセージ | テーマ名と深さ（標準）が含まれる |
| 3 | 調査未開始 | A-2 の時点では進捗ログが動いていない |
| 4 | 承認後に開始 | `はい` 送信後に進捗ログが動き始める |
| 5 | 承認バナー出現 | web_search/web_fetch のたびにバナーが出る |
| 6 | 取り込み反映 | 「取り込む」後に完了ログが出る |
| 7 | スキップ反映 | 「スキップ」後もシステムが止まらず続く |
| 8 | 完了メッセージ | CSM から完了と出力パスが通知される |
| 9 | 出力ファイル生成 | 指定パスに .md ファイルが存在する |
