# AGENTS.md — AI開発エージェントへの指示

このファイルはAIエージェント（Copilot, Claude等）がこのリポジトリで作業する際に従うべきルールを定めます。

---

## 完了条件（絶対厳守）

コード変更・バグ修正・機能追加のいずれであっても、**「完了」「修正しました」「対応済み」と報告する前に、必ず以下を実行すること。**

### 必須ステップ

```
1. pytest tests/unit/ -x -q を実行する
2. 出力を読み、失敗が 0 件であることを目視確認する
3. 確認した出力結果をレスポンスに含める
4. 失敗が1件でもあれば「完了」と言わない
```

### 許可されない報告パターン

以下の言い方は、テスト実行・確認なしに使用してはならない：

- 「修正しました」「対応しました」「直しました」
- 「完了です」「done」「fixed」
- 「動くはずです」「問題ないと思います」

テストを実行した証拠（出力）なしにこれらを使った場合、その報告は虚偽とみなす。

---

## テスト実行コマンド

```bash
# ユニットテスト（通常はこれだけでよい）
pytest tests/unit/ -x -q

# 全テスト
pytest -x -q
```

## 自動検証できないコンポーネント（手動確認が必要）

以下のファイルを変更した場合、ユニットテストが通過しても **「完了」と報告してはならない**。
必ず「手動スモークテストが未実施」と明記し、人間に確認を求めること。

### 対象ファイル

| ファイル | 理由 | 手動確認方法 |
|---|---|---|
| `src/research_team/search/human.py` | 実ブラウザ・人間操作が必要 | 下記スモークテスト手順を参照 |
| `src/research_team/pi_bridge/client.py` | 実際のpi-agentプロセス起動が必要 | 実際にエージェントを起動して動作確認 |
| `src/research_team/pi_bridge/web_search.ts` | Node.jsプロセス・外部通信が必要 | 実際にエージェントを起動して動作確認 |

### HumanSearchEngine スモークテスト手順

変更後、以下を人間が手動で実行・目視確認すること：

```
1. research-team コマンドを起動する
2. ブラウザウィンドウが開くことを確認する
3. 簡単な検索クエリを投げる（例：「テスト」）
4. Google検索ページが表示され、承認UIが機能することを確認する
5. 承認後、検索結果が返ってくることを確認する
```

### AIエージェントへの指示

上記ファイルを変更した場合、レスポンスに以下を必ず含めること：

```
⚠️ 手動確認が必要：[変更したファイル名] を変更しました。
ユニットテストはパスしましたが、実ブラウザ動作の確認は自動化できません。
スモークテスト手順に従って動作を確認してください。
```

## テンプレートファイル変更時の追加確認

`*.template` ファイルを変更した場合、以下も確認すること：

- `str.format()` で使われるプレースホルダー（`{...}`）を追加・変更した場合、既知のキー以外がないか確認する
- 意図しないプレースホルダーは `{{...}}` にエスケープする
- 変更後に必ずユニットテストを実行する
